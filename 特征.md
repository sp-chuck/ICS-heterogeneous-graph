# ICS 节点特征构建任务总结

## 1. 任务目标

为工业控制系统（ICS）中的设备节点构建节点特征向量，用于后续图建模/图神经网络训练。
节点特征需要融合三类属性：

1. **General Process Properties**（一般工艺属性）
2. **Controlling Properties**（控制属性）
3. **Measurement Properties**（测量属性）

最终每个节点的特征向量应由这三类属性拼接而成：

[
x_i = [x_i^{general}; x_i^{control}; x_i^{measurement}]
]

其中，前两类主要来自系统拓扑/元数据，第三类来自时间序列读数分析。

---

## 2. 要实现的属性类别

根据 Table 5，需要实现如下属性：

| Categories                 | Features                                                                  |
| -------------------------- | ------------------------------------------------------------------------- |
| General Process Properties | Device type, Process & Sub-process                                        |
| Controlling Properties     | Controllers performing pooling                                            |
| Measurement Properties     | Periodicity, Period Value, Period Max(Min), Period Entropy Mean(Variance) |

## 3. 各类属性的抽取内容与方式

---

## 3.1 General Process Properties

### 需要抽取的属性

* `device_type`
* `process`
* `sub_process`

### 含义

这类属性描述设备在工业流程中的静态身份和工艺归属，例如：

* 设备是 sensor / actuator / controller 中的哪一类
* 属于哪个工艺阶段
* 属于哪个子过程

### 抽取方式

这部分通常直接从设备元数据或基础拓扑信息中读取。

### 编码方式

建议使用 **one-hot 编码**：

* `device_type` → one-hot
* `process` → one-hot
* `sub_process` → one-hot

### 输出

```python
general_feature = concat(
    one_hot(device_type),
    one_hot(process),
    one_hot(sub_process)
)
```

---

## 3.2 Controlling Properties

### 需要抽取的属性

* `controllers performing pooling`

### 含义

该属性描述当前节点（例如传感器）与哪些控制器存在控制关联，或者位于哪些 controller pooling 范围之内。

本质上，它反映的是：

* 该节点受哪些控制器影响
* 它在控制拓扑中的位置关系

### 抽取方式

需要从控制器-设备关联关系中读取，例如：

* controller -> controlled sensors 映射表
* sensor -> related controllers 映射表

### 编码方式

建议使用 **multi-hot 编码**：

* 若一共有 `M` 个 controller
* 对于当前节点，若第 `j` 个 controller 与其相关，则第 `j` 位设为 1，否则为 0

### 输出

```python
control_feature[j] = 1 if controller_j affects current_node else 0
```

---

## 3.3 Measurement Properties

### 需要抽取的属性

* `Periodicity`
* `Period Value`
* `Period Max`
* `Period Min`
* `Period Entropy Mean`
* `Period Entropy Variance`

### 含义

这类属性来自节点的时间序列读数，描述该序列是否具有周期性，以及在周期尺度下的统计特征。

### 核心思路

不是直接在整条原始序列上计算所有统计量，而是：

1. 先判断序列属于哪种周期类型
2. 若存在周期性，则先通过 **Sequence Period Mining** 找到候选周期 `T`
3. 再基于长度为 `T` 的窗口集合计算周期相关统计量

---

# 4. Measurement Properties 的详细实现逻辑

---

## 4.1 周期性类型 `Periodicity`

### 要区分的三种类型

1. **non-constant periodic**
2. **constant**
3. **non-periodic**

### 编码方式

按论文描述，使用 one-hot：

* periodic → `(1, 0, 0)`
* constant → `(0, 1, 0)`
* non-periodic → `(0, 0, 1)`

### 判定逻辑建议

#### 1）先判断 constant

若序列标准差极小，视为 constant：

```python
if np.std(signal) < constant_threshold:
    type = "constant"
```

#### 2）否则执行 period mining

若通过周期挖掘找到可信周期 `T`，则视为 periodic。

#### 3）若找不到可信周期

则视为 non-periodic。

---

## 4.2 周期 `Period Value` 的获取

### 目标

从时间序列 `S` 中挖掘候选周期 `T`。

### 论文对应方法

使用 **Algorithm 2: Sequence Period Mining**

其核心思想不是 FFT，而是：

> 通过扫描不同窗口长度，寻找使“滑动窗口熵序列的方差”达到局部最小的窗口长度，把该窗口长度视为候选周期。

---

# 5. Sequence Period Mining 的实现说明

## 5.1 输入与输出

### 输入

* 时间序列 `S`
* 参数：

  * `w0`：初始窗口长度
  * `w_max`：窗口长度上界
  * `delta`：窗口长度增长步长
  * `delta_w`：滑动步长

### 输出

* 候选周期 `T`

---

## 5.2 算法思路

对于每个候选窗口长度 `ell`：

1. 以步长 `delta_w` 在序列上滑动
2. 对每个长度为 `ell` 的窗口计算熵
3. 得到一个“窗口熵序列”
4. 计算该熵序列的方差
5. 记录不同 `ell` 下的熵方差曲线
6. 若某个 `ell` 对应的熵方差出现局部最小，并满足“局部凸性/先降后升”趋势，则把该 `ell` 作为周期 `T`

---

## 5.3 数学理解

对每个候选窗口长度 `ell`，构造滑窗集合：

[
P_\ell = {S[\psi:\psi+\ell]}
]

其中 `ψ` 是所有滑窗起点。

然后对每个窗口计算熵：

[
E_1, E_2, \dots, E_n
]

得到熵序列后，计算其方差：

[
\Lambda^{(Var)}(\ell) = Var(E_1, E_2, \dots, E_n)
]

### 判据

* 若 (\Lambda^{(Var)}(\ell)) 较小，说明该尺度下不同窗口的熵更相似
* 若它在某个 `ell` 处达到局部最小，则说明在该尺度下序列最“平稳/重复”
* 因此该 `ell` 很可能就是周期

---

## 5.4 推荐实现方式

建议不要机械照抄伪代码，而是采用更工程化的两层实现：

### 第一层：对每个候选窗口长度 `ell` 计算一个得分

得分就是熵序列方差：

```python
score[ell] = variance(entropies_of_all_windows_with_length_ell)
```

### 第二层：在 `score[ell]` 上找局部最小值

要求：

* 有明显先下降后上升趋势
* 不是随机噪声造成的假极小值

### 推荐输出

* 返回第一个显著局部最小值对应的 `ell`
* 若没有找到，则返回 `w_max` 或判为 non-periodic

---

## 5.5 伪代码

```python
def sequence_period_mining(signal, w0, w_max, delta, delta_w):
    scores = []

    for ell in range(w0, w_max + 1, delta):
        windows = []
        for start in range(0, len(signal) - ell + 1, delta_w):
            windows.append(signal[start:start + ell])

        if len(windows) == 0:
            continue

        entropies = [window_entropy(w) for w in windows]
        var_entropy = variance(entropies)
        scores.append((ell, var_entropy))

    T = find_local_minimum_with_convexity_check(scores)

    if T is None:
        return w_max
    return T
```

# 6. 窗口熵 `window_entropy` 的实现

## 6.1 作用

用于衡量一个滑动窗口内部的“信息复杂度”。

## 6.2 实现建议

可以使用 Shannon entropy：

1. 对窗口内数值做直方图离散化
2. 得到概率分布
3. 计算熵

### 伪代码

```python
def window_entropy(x, bins=16):
    hist, _ = np.histogram(x, bins=bins, density=False)
    prob = hist / np.sum(hist)
    prob = prob[prob > 0]
    return -np.sum(prob * np.log(prob + 1e-12))
```

# 7. 找到周期 `T` 以后，如何提取周期相关属性

设：

* `T` 为周期
* `P` 为所有长度为 `T` 的窗口集合
* `n_P` 为窗口数

即：

```python
windows = [signal[w:w+T] for w in starts]
```

---

## 7.1 Period Max

论文公式：

[
\rho_{\max}=\frac{1}{n_P}\sum_{\forall w\in P} \sup{S[w:w+T]}
]

### 含义

对每个长度为 `T` 的窗口，取最大值，再对这些最大值求平均。

### 实现

```python
rho_max = mean([max(window) for window in windows])
```

---

## 7.2 Period Min

论文公式：

[
\rho_{\min}=\frac{1}{n_P}\sum_{\forall w\in P} \inf{S[w:w+T]}
]

### 含义

对每个长度为 `T` 的窗口，取最小值，再对这些最小值求平均。

### 实现

```python
rho_min = mean([min(window) for window in windows])
```

---

## 7.3 Period Entropy Mean

论文公式：

[
\mu(E_i^{(k)})=\frac{1}{n_P}\sum_{\forall w\in P} E_i^{(k)}
]

### 含义

对每个长度为 `T` 的窗口计算熵，然后求均值。

### 实现

```python
entropies = [window_entropy(window) for window in windows]
mu_e = mean(entropies)
```

---

## 7.4 Period Entropy Variance

论文公式：

[
\sigma^2(E_i^{(k)})=\frac{1}{n_P}\sum_{\forall w\in P} [\mu(E_i^{(k)}) - E_i^{(k)}]^2
]

### 含义

计算所有周期窗口熵相对于熵均值的方差。

### 实现

```python
var_e = mean([(mu_e - e) ** 2 for e in entropies])
```

---

# 8. 不同周期类型下的赋值规则

## 8.1 constant

若序列为常数流：

* periodicity = `(0,1,0)`
* period value = `0`
* period max = 常数值
* period min = 常数值
* entropy mean = `0`
* entropy variance = `0`

---

## 8.2 periodic

若序列为非恒定周期流：

* periodicity = `(1,0,0)`
* period value = `T`
* period max = `rho_max`
* period min = `rho_min`
* entropy mean = `mu_e`
* entropy variance = `var_e`

---

## 8.3 non-periodic

若找不到有效周期：

* periodicity = `(0,0,1)`
* period value = `infinity`（实现中不要直接用 `np.inf`，建议用大数代替，如 `1e6`）
* period max = 全局最大值
* period min = 全局最小值
* entropy mean = 占位值
* entropy variance = 占位值

### 建议

工程实现里对 `infinity` 使用大数替代，比如：

```python
LARGE_VALUE = 1e6
```

否则后续标准化/训练容易出数值问题。

```python
def extract_measurement_features(signal, config):
    if is_constant(signal, config):
        return build_constant_measurement_feature(signal)

    T = sequence_period_mining(signal, config)

    if not is_valid_period(T, signal, config):
        return build_nonperiodic_measurement_feature(signal, config)

    windows = build_windows(signal, T, config["delta_w"])
    entropies = [window_entropy(w) for w in windows]

    rho_max = mean([max(w) for w in windows])
    rho_min = mean([min(w) for w in windows])
    mu_e = mean(entropies)
    var_e = mean([(mu_e - e) ** 2 for e in entropies])

    return concat(
        [1, 0, 0],      # periodic
        [T],
        [rho_max],
        [rho_min],
        [mu_e],
        [var_e]
    )
```

# 12. 实现时的关键注意事项

## 12.1 不要直接在代码里用 `np.inf`

论文里可以写 infinity，但实现中建议用大数占位。

---

## 12.2 `period mining` 是这部分最关键的核心

真正的 measurement feature 质量，最依赖 `T` 是否挖得合理。

---

## 12.3 period mining 的本质

它不是“直接做频谱分析”，而是：

> 扫描不同窗口长度，寻找使窗口熵序列方差最小、最稳定的尺度。

---

## 12.4 最好对连续特征做归一化

例如：

* period value
* rho_max
* rho_min
* entropy mean
* entropy variance

而 one-hot / multi-hot 保持原样。
