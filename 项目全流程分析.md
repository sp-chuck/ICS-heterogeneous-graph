# 深度学习项目全流程分析文档（HTGNN 异构图异常检测）

## 项目概览

本项目面向 SWaT 工业控制场景，使用异构图时序建模进行异常检测。当前工程已完成模块化重构：

- 数据准备与图构建：DataPreprocess_new.py
- 模型定义：models/legacy_htgnn.py 与 models/official_htgnn_core.py
- 训练调度：train_htgnn.py
- 指标评估：utils/evaluation.py

数据流简述：

1. 原始表格数据转为静态异构图与时序快照
2. 快照序列输入 HTGNN 后端（legacy 或 official_core）
3. 训练时支持 anomaly_cls 与 regression 两类任务
4. 在验证集早停，最终在 split-test 或 external-test 上评估

---

## 1. 数据准备

### 1.1 数据集来源与构成

项目使用 SWaT 数据集，核心输入包含：

- train.csv
- test.csv
- list.txt（传感器列表）
- graph100.txt（物理距离矩阵）

根据预处理报告，当前一次典型构建的节点规模与边规模如下：

- 节点：58（device 51, plc 6, crp 1）
- 边：970（hierarchy 114, physical 742, flow 114）
- 孤立节点：0

### 1.2 预处理流程

预处理入口为 DataPreprocess_new.py，主要阶段：

1. 列对齐与数据清洗
2. 节点 schema 构建（CRP/PLC/device 三层）
3. 节点特征构建（v2 或 v3）
4. 三类边构建
   - hierarchy：CRP-PLC、PLC-device
   - physical：基于 graph100.txt 与阈值
   - flow：基于规则与周期特征相似度
5. 导出静态图 numpy 表示
6. 可选导出时序快照

### 1.3 时序快照与标签生成

时序快照参数来自预处理配置：

- temporal_window = 256
- temporal_stride = 128

标签规则：窗口内只要存在 attack 行，则 snapshot_label=1。

本次结果（来自 snapshots_metadata.json）：

- 训练快照目录 temporal_snapshots：370 个快照，normal 370，attack 0
- 测试快照目录 temporal_snapshots_test：350 个快照，normal 260，attack 90

这意味着训练集是单类正常样本，属于典型异常检测设定，因此训练脚本中引入了基于训练分位数的阈值策略。

### 1.4 数据加载方式（训练阶段）

train_htgnn.py 的加载逻辑：

- load_one_snapshot
  - 读取每个快照目录下的 x_crp.npy, x_plc.npy, x_device.npy
  - 读取 hetero_meta.json 中的 relations 并加载 edge_index 与 edge_weight
  - 读取 snapshot_meta.json 获得 snapshot_label
  - 可选读取 device_ground_truth.npy/csv 作为辅助监督
- load_snapshots
  - 汇总全体快照
  - 合并关系类型集合 relation_list
- make_sequences
  - 将连续快照切分为长度 seq_len 的序列
- chronological_split
  - 按时间顺序拆分 train/val/test

## 2. 模型架构

### 2.1 双后端设计

项目支持两个后端，通过 train_htgnn.py 的 backend 参数选择：

1. legacy_pyg（models/legacy_htgnn.py）
2. official_htgnn_core（models/official_htgnn_core.py）

### 2.2 legacy_pyg 架构

结构：SnapshotEncoder + GRU + 任务头

- SnapshotEncoder
  - 每类节点 LazyLinear 投影
  - HeteroConv（SAGEConv）进行单快照关系聚合
  - 各节点类型池化后拼接为图级向量
- 时间建模
  - GRU 对快照序列进行时序编码
- 输出头
  - cls：图级输出（异常分类或回归）
  - device_cls：设备级辅助输出

设计动机：实现简单、稳定、可作为基线。

### 2.3 official_htgnn_core 架构

结构：RelationAgg + TemporalAgg + 任务头

- OfficialCoreSnapshotEncoder
  - 针对每种关系使用 SAGEConv
  - 使用 RelationAgg 对多关系消息进行注意力式聚合
- TemporalAgg
  - 使用位置编码 + Q/K/V 注意力建模时间维
- 输出头
  - cls 与 device_cls 与 legacy 保持一致

设计动机：对齐官方 HTGNN 思路，在关系维与时间维都引入更显式聚合机制，提高结构表达能力。

### 2.4 激活函数、优化器与正则

- 激活函数：ReLU
- Dropout：用于节点投影与关系聚合后表示
- 优化器：Adam
- Weight decay：默认 1e-5

## 3. 训练过程

### 3.1 训练策略

训练入口 train_htgnn.py 的核心策略如下：

1. 设定随机种子（Python, NumPy, Torch）
2. 按时间顺序划分 train/val/test
3. 每个 epoch 对 train 与 val 分别执行 run_epoch
4. 早停策略：patience 次不提升后停止
5. 保存最佳 checkpoint
6. 在最终测试集上评估

### 3.2 损失函数

由 task_mode 控制：

- anomaly_cls
  - snapshot loss：BCEWithLogitsLoss
  - device aux loss：BCEWithLogitsLoss
  - 总损失：snapshot + device_loss_alpha × device
- regression
  - snapshot loss：L1 或 MSE
  - device aux loss 可选保留

### 3.3 训练流程细节

run_epoch 的流程：

1. 取序列最后一个快照标签作为监督目标
2. 前向得到 graph logit 与 device logit
3. 计算主损失与辅助损失
4. 若训练模式则反向更新
5. 收集 batch 级分数与标签
6. 调用 utils/evaluation.py 的 summarize_epoch_metrics 做指标汇总

### 3.4 训练时间记录

当前日志包含 epoch 进度与早停，但未记录每个 epoch 耗时。若需要时间统计，可在命令前加系统计时：

```bash
time python train_htgnn.py --backend official_htgnn_core --task_mode anomaly_cls
```

## 4. 评估与测试

### 4.1 评估模块

评估逻辑集中在 utils/evaluation.py：

- summarize_epoch_metrics：epoch 内汇总分类/回归与设备级指标
- resolve_threshold：fixed/train_quantile/auto 三种阈值
- evaluate_with_threshold：输出阈值化分类指标与混淆矩阵项
- evaluate_regression：输出 MAE/MSE/RMSE
- collect_scores：提取 train/test 序列分数用于阈值估计

### 4.2 使用指标

anomaly_cls：

- loss, acc, precision, recall, f1
- auc, auprc
- tn, fp, fn, tp
- device_acc, device_precision, device_recall, device_f1, device_auc

regression：

- loss, mae, mse, rmse
- 可选 device 指标

### 4.3 测试集评估方式

- 默认测试：时间切分得到 split-test
- 外部测试：当提供 test_data_dir 时，使用 external-test

当前日志显示已使用 external-test（temporal_snapshots_test）。

## 5. 实验结果

以下结果来自现有日志 htgnn_best.log（同一数据准备配置）：

### 5.1 数据分布背景

- 训练序列标签：全为 normal（attack=0）
- 验证序列标签：全为 normal（attack=0）
- 外部测试集包含攻击样本

因此直接用 0.5 固定阈值时，模型倾向预测全 normal，阈值后处理尤为关键。

### 5.2 legacy_pyg 后端（anomaly_cls）

- 训练早停：epoch 11
- external-test 原始阈值（0.5）
  - acc=0.7580, precision=0.0000, recall=0.0000, f1=0.0000
  - auc=0.6345, auprc=0.5974
- 使用 auto 阈值后
  - threshold=0.000022
  - acc=0.6064, precision=0.3219, recall=0.5663, f1=0.4105
  - tn=161, fp=99, fn=36, tp=47

### 5.3 official_htgnn_core 后端（anomaly_cls）

- 训练早停：epoch 11
- external-test 原始阈值（0.5）
  - acc=0.7580, precision=0.0000, recall=0.0000, f1=0.0000
  - auc=0.6113, auprc=0.5069
- 使用 auto 阈值后
  - threshold=0.000000
  - acc=0.3848, precision=0.2594, recall=0.8313, f1=0.3954
  - tn=63, fp=197, fn=14, tp=69

### 5.4 结果解读

- 两个后端在单类训练场景下都依赖阈值后处理才能召回攻击。
- legacy_pyg 在本次日志中 AUC/AUPRC 略高。
- official_core 在 auto 阈值下召回更高，但误报也更高。

## 6. 附加内容

### 6.1 关键超参数与技巧

- 阈值策略
  - decision_mode=auto 对单类训练场景尤为重要
  - train_score_quantile 可用于控制召回-误报平衡
- 设备辅助监督
  - device_loss_alpha 可增强 device 级监督信号
- 时间切分
  - chronological_split 避免时间泄漏
- 模块化重构
  - 模型与评估已分离，便于后续替换 trainer 或新增后端

### 6.2 可视化建议（训练曲线与混淆矩阵）

可以基于日志文件直接生成图表，示例代码如下：

```python
import re
import matplotlib.pyplot as plt
from sklearn.metrics import ConfusionMatrixDisplay

log_path = "htgnn_best.log"
text = open(log_path, "r", encoding="utf-8").read().splitlines()

epochs, train_loss, val_loss = [], [], []
for line in text:
    m = re.search(r"Epoch\s+(\d+)\s+\|\s+train_loss\s+([0-9.]+).*val_loss\s+([0-9.]+)", line)
    if m:
        epochs.append(int(m.group(1)))
        train_loss.append(float(m.group(2)))
        val_loss.append(float(m.group(3)))

plt.figure(figsize=(8,4))
plt.plot(epochs, train_loss, label="train_loss")
plt.plot(epochs, val_loss, label="val_loss")
plt.xlabel("epoch")
plt.ylabel("loss")
plt.title("Training Curves")
plt.legend()
plt.tight_layout()
plt.show()

# 例：从日志中的 tn/fp/fn/tp 手动填入
tn, fp, fn, tp = 161, 99, 36, 47
cm = [[tn, fp], [fn, tp]]
ConfusionMatrixDisplay(confusion_matrix=cm, display_labels=["normal", "attack"]).plot(cmap="Blues")
plt.title("Confusion Matrix (thresholded)")
plt.tight_layout()
plt.show()
```

### 6.3 可改进方向

1. 引入 attack 样本参与训练或半监督策略，缓解单类训练偏置。
2. 对 auto 阈值加入验证集约束，降低误报。
3. 记录 epoch 耗时与更多过程指标（如 PR 曲线点）。
4. 增加多次随机种子重复实验，报告均值与方差。
