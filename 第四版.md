# 第四版（工程化重构完成）

## 1. 版本目标

第四版聚焦训练工程结构重构：

- 将官方核心 HTGNN 实现从训练入口中剥离
- 将 legacy PyG 实现也剥离到独立模型模块
- 将评估与阈值决策逻辑抽离为独立工具模块
- 让训练入口脚本只保留调度、训练循环与日志编排

---

## 2. 目录与模块变化

### 2.1 新增模块

- models/official_htgnn_core.py
  - RelationAgg
  - TemporalAgg
  - OfficialCoreSnapshotEncoder
  - OfficialCoreHTGNN

- models/legacy_htgnn.py
  - SnapshotEncoder
  - LegacyHTGNN

- utils/evaluation.py
  - collect_scores
  - evaluate_regression
  - evaluate_with_threshold
  - resolve_threshold
  - summarize_epoch_metrics

- models/__init__.py
- utils/__init__.py

### 2.2 入口脚本职责收敛

- train_htgnn.py
  - 保留：参数解析、数据加载、训练循环、早停、checkpoint、日志、测试阶段调度
  - 抽离：模型定义、指标汇总、阈值与评估细节

---

## 3. 可选后端与任务模式

### 3.1 后端

- legacy_pyg
  - 来自 models/legacy_htgnn.py
  - 结构：HeteroConv/SAGE + GRU

- official_htgnn_core
  - 来自 models/official_htgnn_core.py
  - 结构：RelationAgg + TemporalAgg（官方风格核心）

### 3.2 任务模式

- anomaly_cls
  - 快照二分类（BCE）
  - 支持阈值策略：fixed / train_quantile / auto

- regression
  - 连续值回归（L1 或 MSE）
  - 输出 MAE/MSE/RMSE

## 4. 全部运行指令

以下命令默认在项目根目录执行：/data/spz/AHGA

### 4.1 进入目录

```bash
cd /data/spz/AHGA
```

### 4.2 环境快速检查

```bash
python -V
python -m py_compile train_htgnn.py models/legacy_htgnn.py models/official_htgnn_core.py utils/evaluation.py
```

### 4.3 数据预处理（静态图 + 时序快照）

```bash
python DataPreprocess_new.py \
  --data_dir /data/spz/data/swat_whole \
  --out_dir /data/spz/AHGA \
  --physical_threshold 15 \
  --feature_version v3 \
  --period_w0 16 \
  --period_w_max 256 \
  --period_delta 4 \
  --period_delta_w 2 \
  --entropy_bins 16 \
  --constant_threshold 1e-6 \
  --large_period_value 1e6 \
  --period_convexity_drop_ratio 0.02 \
  --period_min_autocorr 0.3 \
  --temporal_mode \
  --temporal_window 256 \
  --temporal_stride 128
```

### 4.4 同时构建测试集时序快照（可选）

```bash
python DataPreprocess_new.py \
  --data_dir /data/spz/data/swat_whole \
  --out_dir /data/spz/AHGA \
  --physical_threshold 15 \
  --feature_version v3 \
  --temporal_mode \
  --build_test_temporal \
  --test_temporal_subdir temporal_snapshots_test
```

### 4.5 训练：legacy 后端 + 异常检测（默认推荐对照基线）

```bash
python train_htgnn.py \
  --data_dir /data/spz/AHGA/temporal_snapshots \
  --backend legacy_pyg \
  --task_mode anomaly_cls \
  --decision_mode auto \
  --save_path /data/spz/AHGA/htgnn_legacy_cls.pt
```

### 4.6 训练：官方核心后端 + 异常检测（本版重点）

```bash
python train_htgnn.py \
  --data_dir /data/spz/AHGA/temporal_snapshots \
  --backend official_htgnn_core \
  --task_mode anomaly_cls \
  --decision_mode auto \
  --save_path /data/spz/AHGA/htgnn_official_core_cls.pt
```

### 4.7 训练：官方核心后端 + 回归

```bash
python train_htgnn.py \
  --data_dir /data/spz/AHGA/temporal_snapshots \
  --backend official_htgnn_core \
  --task_mode regression \
  --regression_loss l1 \
  --save_path /data/spz/AHGA/htgnn_official_core_reg.pt
```

### 4.8 使用外部测试快照目录评估（可选）

```bash
python train_htgnn.py \
  --data_dir /data/spz/AHGA/temporal_snapshots \
  --test_data_dir /data/spz/AHGA/temporal_snapshots_test \
  --backend official_htgnn_core \
  --task_mode anomaly_cls \
  --decision_mode train_quantile \
  --train_score_quantile 0.95 \
  --save_path /data/spz/AHGA/htgnn_official_core_cls_external.pt
```

### 4.9 常用阈值策略示例

```bash
# 固定阈值
python train_htgnn.py --backend official_htgnn_core --task_mode anomaly_cls --decision_mode fixed --fixed_threshold 0.5

# 训练分位数阈值
python train_htgnn.py --backend official_htgnn_core --task_mode anomaly_cls --decision_mode train_quantile --train_score_quantile 0.95
```

### 4.10 快速 smoke（1 epoch）

```bash
python train_htgnn.py \
  --data_dir /data/spz/AHGA/temporal_snapshots \
  --backend official_htgnn_core \
  --task_mode anomaly_cls \
  --epochs 1 \
  --patience 1 \
  --save_path /data/spz/AHGA/htgnn_smoke.pt
```

---

## 5. 关键参数说明

- backend: legacy_pyg | official_htgnn_core
- task_mode: anomaly_cls | regression
- regression_loss: l1 | mse
- decision_mode: auto | fixed | train_quantile
- device_loss_alpha: 设备辅助监督损失权重，0 表示关闭
- test_data_dir: 指向外部时序快照目录，存在则优先作为最终测试集

---

## 6. 结果文件位置

- 训练模型：save_path 指定的 .pt 文件
- 日志文件：默认与 save_path 同名 .log
- 静态图与快照：
  - /data/spz/AHGA/static_graph

  - /data/spz/AHGA/temporal_snapshots_test

## 7. 第四版结论

第四版完成了从“脚本堆叠式实现”到“模块化训练框架”的迁移：

- 模型层、评估层、调度层职责分离
- 保留 legacy 兼容路径，支持安全回退
- 提供官方核心后端开关与任务模式切换
- 后续可继续将 run_epoch 拆分到 trainer 模块，进一步简化入口脚本
