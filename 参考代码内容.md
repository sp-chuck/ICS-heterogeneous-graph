# 参考代码内容

## 1. 代码当前实现的核心功能

项目整体是一个基于图神经网络的工业时序异常检测流程，分为三段：

1. 数据预处理（DataPreprocess_Data.py）
2. 相关关系学习（AHGA(CorrelationAnalyzer).py）
3. 异常检测分类（AGHA(Anomaly Detector).py）

对应关系是：
原始时序数据 -> 节点特征构建 -> 边关系学习 -> 图分类异常检测。

## 2. 各文件功能说明

### 2.1 DataPreprocess_Data.py（预处理与特征构图）

该脚本实现了从原始工业控制数据到图节点特征文件的完整预处理，关键能力如下。

1. 滑动窗口指数加权特征提取
- `window_size=4300`，`balance_factor=4`。
- 对每个传感器列使用指数衰减权重做窗口内加权聚合（近时刻权重大）。
- 输出每个设备的特征文件：`Preprocessed_Features_Device_*.csv`。

2. 滑动窗口标签生成
- 标签列索引为 `label_index_in=52`。
- 窗口内只要出现攻击样本即标记为 1，否则为 0。
- 输出：`Preprocessed_Labels.csv`。

3. 下采样与非线性压缩
- 每 10 行取 1 行（10:1 下采样）。
- 对特征做 `atan(x * s)` 变换（标签不变）。
- 输出：
  - `Preprocessed_Downsampled_Features_Device_*.csv`
  - `Preprocessed_Downsampled_Labels.csv`

4. 样本打乱与多节点拼接
- 将时间点随机打乱，按每个时间点构建 43 节点结构（1 CRP + 6 控制器 + 35 设备 + 1 标签节点）。
- 输出节点文件：`swat_nodes_3_all_time_ticks_dev_alt.csv`。

5. PCA 降维
- 对拼接后的高维节点特征做 PCA，保留 99% 信息。
- 输出：`swat_nodes_3_pca_all_time_ticks_dev_alt.csv`。

6. 统计工具函数
- `count_labels_original()`：统计原始数据集中异常比例。
- `count_labels()`：统计预处理后节点文件中的异常比例。

当前脚本结尾默认执行：
- `sample_shuffle()`
- `PCA_com()`

其余步骤函数（如 `feature_processing()`、`label_processing()`、`down_sampling()`）处于注释状态，需要手动放开。

### 2.2 AHGA(CorrelationAnalyzer).py（关联分析/边学习）

该脚本使用 PyTorch Geometric 的 GCN 做链路预测（Link Prediction），从初始图结构中学习更强的相关边。

1. 输入
- 节点特征：`node.csv`
- 初始边：`link.csv`

2. 模型结构
- `GCNConv` 多层编码器（含 LeakyReLU + Dropout）。
- `decode` 用节点表示点积预测边存在概率。
- `decode_all` 生成全图候选边。

3. 训练与评估
- 使用 `RandomLinkSplit` 划分 train/val/test，并自动构造正负边样本。
- 指标：Recall、Precision、F1、Accuracy、AUC。

4. 输出
- 测试阶段调用 `decode_all`，并写出学习到的边文件：`swat_edge_index_dev_alt.csv`。

### 2.3 AGHA(Anomaly Detector).py（异常检测）

该脚本实现最终异常分类模型（自定义 GCN 层 + 全连接分类头）。

1. 数据加载与图构建
- `load_data(path, dataset, elements)` 按前缀读取：
  - 节点文件：`{dataset}_{elements[0]}.csv`
  - 边文件：`{dataset}_{elements[1]}.csv`
- 当前默认参数：
  - `dataset="swat"`
  - `elements=["nodes_3_pca_all_time_ticks_dev_alt", "edges_learnt_dev_alt"]`

2. 图处理
- 构建稀疏邻接矩阵并做对称归一化。
- 将样本按 batch 切分并划分 train/val/test 索引区间。

3. 模型与训练
- 三层图卷积结构（中间层重复调用）+ `Linear` 分类。
- 损失函数：`NLLLoss`。
- 优化器：Adam。

4. 测试指标
- Accuracy、Recall、Precision、F1、AUROC、AUPRC。

## 3. SWAT 与 WADI 实验适配现状

### 已具备能力

1. SWAT：基本全流程可跑
- 预处理脚本中原始数据文件名、输出文件名均为 SWAT 命名。
- 关联分析与检测脚本默认使用 SWAT 文件命名。

2. WADI：检测脚本具备参数化入口
- `AGHA(Anomaly Detector).py` 的 `load_data()` 支持通过 `dataset` 参数切换前缀，理论可读取 WADI 对应文件。

### 当前限制

1. 预处理脚本强绑定 SWAT
- 原始输入硬编码为 `SWaT_Dataset_Attack_v0.xlsx`。
- 输出文件也固定为 `swat_*` 命名。

2. 关联分析脚本输出固定 SWAT 文件名
- 固定写出 `swat_edge_index_dev_alt.csv`。

3. 检测脚本默认边文件名与现有输出不一致
- 默认 `elements[1]="edges_learnt_dev_alt"`。
- 当前仓库存在和相关分析脚本输出的是 `swat_edge_index_dev_alt.csv`。
- 因此实际运行时通常需要统一命名或改 `elements` 参数。

## 4. 当前代码对应的实验流水线（按文件）

1. DataPreprocess_Data.py
- 从原始时序数据提取窗口特征与标签，构建节点并降维。

2. AHGA(CorrelationAnalyzer).py
- 基于初始结构学习相关边，生成边索引文件。

3. AGHA(Anomaly Detector).py
- 读取节点与边，进行图异常分类并输出多项指标。

## 5. 结论

当前代码已经实现了 SWAT 实验的完整三阶段方案（预处理、关系学习、异常检测）。

对于 WADI，异常检测模块已经有参数化读取机制，但预处理与边学习脚本仍以 SWAT 文件名和数据格式为主，需要做文件命名和输入路径层面的适配后，才能形成同等完整的 WADI 端到端流程。
