# 异构图选取与处理全流程

本文说明当前代码中采用的异构图建模方式，以及从原始 SWAT 数据到静态图、时序快照、再到 HTGNN 训练的完整处理链路。对应的核心实现主要在 [DataPreprocess_new.py](DataPreprocess_new.py) 和 [train_htgnn.py](train_htgnn.py) 中。

## 1. 为什么选择异构图

SWAT 场景里的对象不是单一类型节点，而是天然分层的工业控制系统结构：

- CRP：虚构的顶层聚合节点
- PLC：控制层节点
- Device：现场设备节点，对应 `list.txt` 和 `train.csv` 中的传感器列

同时，节点之间也不是单一类型关系，而是存在多种语义明确的边：

- 层级关系：CRP -> PLC -> Device
- 物理关系：设备之间的空间/物理邻接
- 流量关系：设备之间在正常工况下的统计相关性

如果把这些信息压成普通同构图，会丢失“节点类型”和“边类型”的语义。代码因此采用三层异构图，把结构关系、物理关系和数据关联关系分开编码，再交给异构 GNN 处理。

## 2. 输入数据与预处理入口

预处理入口在 [DataPreprocess_new.py](DataPreprocess_new.py)。默认读取：

- `train.csv`：训练数据
- `test.csv`：测试数据，可选导出测试时序快照
- `list.txt`：设备/传感器名称列表
- `graph100.txt`：物理距离矩阵

核心参数包括：

- `--physical_threshold`：物理边阈值
- `--corr_threshold`：流量边阈值
- `--temporal_mode`：是否导出时序快照
- `--build_test_temporal`：是否对 `test.csv` 额外导出时序快照
- `--feature_version`：节点特征版本，`v2` 或 `v3`
- `--legacy_window_mode`：是否使用旧版块聚合特征逻辑

## 3. 节点如何构建

### 3.1 三层节点 schema

代码在 `build_node_schema()` 中构造固定的三层节点：

- 1 个 CRP 节点：`CRP0`
- 6 个 PLC 节点：`PLC1` 到 `PLC6`
- 多个 Device 节点：由 `list.txt` 与 `train.csv` 对齐得到

每个节点都带有：

- `global_id`
- `node_name`
- `node_type`
- `plc_name`
- `plc_index`

这一步的作用是把工业系统中的实体统一映射到一个全局索引空间，后续边和特征都按这个映射落盘。

### 3.2 设备到 PLC 的分配规则

设备节点会被分配到某个 PLC，规则在 `infer_plc_index()` 中完成：

- 先提取设备名中的数字
- 取数字首位作为 PLC 编号
- 例如 `1xx -> PLC1`，`2xx -> PLC2`，直到 `6xx -> PLC6`

如果设备名不满足这个规则，则默认归到 `PLC1`。

这个映射把底层设备和控制层 PLC 建立起显式的层级语义，是后面构建 hierarchy 边的基础。

## 4. 节点特征如何生成

### 4.1 设备特征

代码会先筛出训练集中 `attack == 0` 的正常样本，再对每个设备列做特征提取。

当 `feature_version=v3` 且未启用 `legacy_window_mode` 时，走 `ics_feature_v3.py` 中的 v3 特征构造逻辑，返回更完整的语义特征；否则走旧版统计特征逻辑。

旧版特征主要包括：

- `mean`
- `std`
- `min`
- `max`
- `last`
- `slope`
- `wmean`

其中 `wmean` 使用指数衰减权重，更重视最近时刻的数据。

如果开启 `legacy_window_mode`，则对窗口内数据做分块加权聚合，得到更长维度的块特征，这与旧版 `DataPreprocess_Data.py` 的思想一致。

### 4.2 PLC 和 CRP 特征

设备特征得到后，代码继续向上聚合：

- 同一 PLC 下的设备特征做平均，形成 PLC 特征
- 所有 PLC 特征再做平均，形成 CRP 特征

也就是说，节点特征不是分别独立训练出来的，而是按系统层级自底向上聚合得到。

### 4.3 全局归一化

`build_node_features()` 最后会对所有节点的每个特征列做 z-score 标准化：

- 均值减去全局均值
- 再除以标准差
- 如果标准差为 0，则直接置为 0

这样可以让不同节点类型的特征处在可比较的数值尺度上。

## 5. 边如何构建

代码一共构建三类边，分别对应三种语义。

### 5.1 hierarchy 边

在 `build_hierarchy_edges()` 中生成：

- `CRP0 <-> PLC1..PLC6`
- `PLCi <-> 其下属 device`

边权重固定为 `1.0`。

这类边表达的是“系统组织结构”，是最基础的拓扑骨架。

### 5.2 physical 边

在 `build_physical_edges()` 中读取 `graph100.txt` 距离矩阵，按阈值建边：

- 若两个设备之间距离 `d <= physical_threshold`，则连物理边
- 权重计算为 `1 - d / threshold`
- 距离越近，权重越大

这类边表示现场设备之间的空间/物理关联。

### 5.3 flow 边

在 `build_flow_edges()` 中基于正常样本的 Pearson 相关系数建边：

- 先对正常样本的设备列做相关矩阵
- 若 `|r| >= corr_threshold`，则连流量边
- 边权重直接取相关系数 `r`

这类边表达的是工况下变量之间的统计耦合关系。

### 5.4 边去重

三类边合并后会经过 `deduplicate_edges()`：

- 以 `(src_global, dst_global, edge_type)` 作为键
- 若重复出现，保留绝对权重更大的那条

这样可以避免多处生成同一类关系时出现重复边。

## 6. 异构图如何落盘

### 6.1 静态图输出

静态图写入 `out_dir/static_graph/`，主要文件包括：

- `node_mapping.csv`
- `hetero_node_features.csv`
- `edges_hierarchy.csv`
- `edges_physical.csv`
- `edges_flow.csv`
- `hetero_edges.csv`
- `x_crp.npy`
- `x_plc.npy`
- `x_device.npy`
- `edge_index__*.npy`
- `edge_weight__*.npy`
- `hetero_meta.json`

其中 `save_numpy_hetero()` 会把不同类型节点拆成独立的特征矩阵，把不同关系拆成独立的 `edge_index` 和 `edge_weight`。

`hetero_meta.json` 会记录：

- 节点类型及数量
- 特征列名
- 关系类型列表
- 对应的边文件名

这套结构是给 PyG 的 `HeteroData` 直接读取的。

### 6.2 可选同构视图

如果开启 `--export_homo_view`，还会额外导出：

- `homograph_nodes.csv`
- `homograph_edges.csv`

这只是辅助查看，不是主训练格式。

## 7. 时序异构图如何处理

当开启 `--temporal_mode` 时，代码会把训练集切成一系列滑动窗口快照。

### 7.1 快照切分

`make_snapshot_ranges()` 使用以下参数：

- `temporal_window`：每个快照包含多少行
- `temporal_stride`：窗口滑动步长
- `max_snapshots`：最多生成多少个快照，0 表示不限制

窗口是按时间顺序滑动的，所以每个 snapshot 都对应一段连续时间片。

### 7.2 快照标签

每个窗口会生成一个图级标签：

- 窗口内只要出现任意 `attack > 0`，`snapshot_label = 1`
- 否则 `snapshot_label = 0`

也就是说，快照级监督是由窗口内是否包含攻击事件决定的。

### 7.3 每个快照的图构建

`run_temporal_snapshots()` 对每个窗口重复执行一遍图构建逻辑：

- 取当前窗口数据
- 仅用窗口内正常样本计算设备特征和 flow 边
- 复用全局 hierarchy 边和 physical 边
- 导出该窗口对应的一整套异构图文件

每个快照目录中会包含：

- `node_mapping.csv`
- `hetero_node_features.csv`
- `edges_flow.csv`
- `hetero_edges.csv`
- `x_*.npy`
- `edge_index__*.npy`
- `edge_weight__*.npy`
- `snapshot_meta.json`
- `device_ground_truth.csv`
- `device_ground_truth.npy`

### 7.4 仅设备节点的监督标签

时序快照还会导出设备级真值：

- 只给 `device` 节点生成 `device_ground_truth`
- `crp` 和 `plc` 不作为监督目标

这对应训练脚本里的辅助任务，用来增强设备级异常识别能力。

### 7.5 测试集时序快照

如果同时开启 `--build_test_temporal`，代码会对 `test.csv` 再生成一套独立快照：

- 默认输出到 `temporal_snapshots_test/`
- 目录结构和训练时序快照一致
- 便于做独立评估或离线推理

## 8. 训练阶段如何消费这些异构图

训练逻辑在 [train_htgnn.py](train_htgnn.py) 中。

### 8.1 读取快照

`load_one_snapshot()` 会把每个快照目录重新组装成 PyG `HeteroData`：

- 从 `x_crp.npy`、`x_plc.npy`、`x_device.npy` 读取节点特征
- 从 `hetero_meta.json` 读取关系定义
- 从 `edge_index__*.npy` 和 `edge_weight__*.npy` 读取边
- 从 `snapshot_meta.json` 读取图级标签
- 从 `device_ground_truth.npy` 读取设备级标签

### 8.2 异构空间编码

`SnapshotEncoder` 的处理方式是：

- 先对不同节点类型做线性投影到统一隐藏维度
- 再用 `HeteroConv + SAGEConv` 按关系类型做消息传递
- 最后对每种节点类型做平均池化
- 将各类型池化结果拼接成一个快照表示

这一步保留了不同节点类型和不同关系的语义差异。

### 8.3 时序建模

`HTGNN` 会把多个快照嵌入按时间顺序送入 GRU：

- 每个快照先得到一个图表示
- 多个快照组成序列
- GRU 建模跨时间演化
- 序列最后状态用于输出快照级二分类结果

### 8.4 辅助设备级监督

训练时还会对最后一个快照的设备表示做辅助 BCE 损失：

- 主任务：快照级是否异常
- 辅任务：设备级是否异常

这种设计让模型既学到图级异常，也保留对底层设备节点的判别能力。

## 9. 全流程总结

代码中的异构图处理可以概括为下面这条链路：

1. 读取 `train.csv`、`test.csv`、`list.txt`、`graph100.txt`
2. 对齐设备列，建立 CRP-PLC-Device 三层节点 schema
3. 按设备名规则映射 PLC
4. 基于正常样本提取设备特征，再向上聚合得到 PLC 和 CRP 特征
5. 构建三类边：hierarchy、physical、flow
6. 合并、去重并导出为 PyG 可读取的异构图文件
7. 在时序模式下按滑动窗口生成多个快照，并为每个快照生成图级标签和设备级标签
8. 训练时把快照恢复为 `HeteroData`，先做异构空间编码，再用 GRU 建模时序演化
9. 用快照级主损失 + 设备级辅助损失完成异常检测训练

## 10. 当前实现的注意点

- 如果窗口内全是攻击样本，代码会退回使用整窗数据计算特征和相关性，避免空正常集导致失败。
- 如果生成的快照里没有攻击标签，可通过 `--fail_on_no_attack_snapshots` 直接中止，避免训练到单类别数据。
- `train_htgnn.py` 依赖 `torch_geometric`，环境里没有安装会直接报错。

## 12. 推荐使用方式

如果你想先生成静态异构图，再生成时序快照，可以直接运行：

```bash
python /data/spz/AHGA/DataPreprocess_new.py \
  --physical_threshold 15 \
  --corr_threshold 0.75 \
  --temporal_mode \
  --temporal_window 256 \
  --temporal_stride 128 \
  --data_dir /data/spz/data/swat_whole \
  --out_dir /data/spz/AHGA
```

如果还需要测试集时序快照：

```bash
python /data/spz/AHGA/DataPreprocess_new.py \
  --physical_threshold 15 \
  --corr_threshold 0.75 \
  --temporal_mode \
  --build_test_temporal \
  --data_dir /data/spz/data/swat_whole \
  --out_dir /data/spz/AHGA
```
