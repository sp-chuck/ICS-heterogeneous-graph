# 异构图构建说明 (Heterogeneous Graph Construction)

本文档基于 `DataPreprocess_new.py` 脚本，详细阐述水处理安全数据集（如SWaT）的三层异构图构建逻辑，包含节点层级定义、特征处理机制、以及三种核心关联边的构建规则。

## 1. 图整体拓扑 (Graph Topology)
构建的图是一个**3层树状异构图 (3-layer Heterogeneous Graph)**，反映了工业控制系统（ICS）自顶向下的物理与逻辑架构。包含三类节点：
- **CRP (Central Room Process)**: 中央控制级，最高抽象层（全局仅1个节点）。
- **PLC (Programmable Logic Controller)**: 可编程逻辑控制器，负责各工艺阶段的区域控制（6个节点，PLC1-PLC6）。
- **Device (设备)**: 现场的传感器（Sensors）与执行器（Actuators），处于最底层，数量由数据维度决定。

## 2. 节点处理与特征构建 (Nodes & Features)

### 2.1 节点映射关系
- **Device 到 PLC 的归属**: 通过解析设备名称中的百位数字（或首个数字）进行匹配。例如带有 `1xx`（如 FIT101）的设备归属于 `PLC1`。
- **PLC 到 CRP 的归属**: 所有 PLC1~PLC6 统一直接归属于虚拟节点 `CRP0`。

### 2.2 特征计算逻辑 (Node Features)
节点特征在时间序列滑窗中被提取，并支持不同版本的特征提取工程 (`v2` 或 `v3` 版)：
1. **Device 特征**:
   - `v2` (Legacy): 对滑窗内数据计算统计量或加权数据块。
   - `v3` (增强版本): 提取工业设备的元数据信息（设备子类型、流程阶段等），并结合时间序列数值特征（周期性、常数特征、熵、最值、方差等）。对于序列中较新的时间点通过 `balance_factor` 施加指数衰减权重(recency bias)。
2. **PLC 特征**:
   - 由物理连接到该 PLC 的所有 Device 的特征向量**取平均值 (Mean Pooling)** 得到。
3. **CRP 特征**:
   - 由所有 PLC (1-6) 的特征向量**取平均值**得到。
4. **特征归一化**:
   - 对所有类型节点的特征进行拼接后，按特征列进行全局的 **Z-score 标准化**，从而消除量纲差异。

## 3. 边处理与构图规则 (Edges Construction)

图中包含三种边类型 (`hierarchy`, `physical`, `flow`)，分别刻画了从控制层级到物理空间分布，再到工业管道工艺流的关联关系。

### 3.1 层级边 (`hierarchy`)
定义了系统控制关系的纵向连接，权重统一为 `1.0`。
- **CRP -> PLC**: `CRP0` 节点连接至全部 6 个 `PLC` 节点。
- **PLC -> Device**: 每个 `PLC` 节点连接至在其管辖范围内的下属 `Device`。

### 3.2 物理边 (`physical`)
仅存在于 `Device -> Device` 之间。刻画现场设备在真实物理空间中的距离邻近性。
- **依据矩阵**: 引入一个预定义的物理距离矩阵文件（如 `graph100.txt`）。
- **建立条件**: 当两个设备节点计算出的距离 $\le$ 给定的阈值 (`physical_threshold`) 时，建立边。
- **权重计算**: 距离越近权重越高。基于最大阈值进行线性缩放归一化到 `[0, 1]` 范围：$w = \max(0, 1.0 - \frac{distance}{threshold})$。

### 3.3 逻辑流边 (`flow`)
仅存在于 `Device -> Device` 之间。刻画工业流体动力学工艺和专家规则形成的隐性连接，权重统一为 `1.0`。基于以下复合规则触发建立：
1. **[基础否决规则]**: 分析仪 (AIT) 拥有高独立性，**不可**与其他任何非AIT类型传感器产生 flow 边。
2. **跨段流量规则**: 相邻阶段（Stage编号相差1）的流量计 (FIT) 相互连接。
3. **同段液位流量规则**: 同一阶段内，流量计(FIT) 和 液位计(LIT/LT) 具有强工艺耦合，相互连接。
4. **管道相邻规则 (启发式)**: 同一工艺段内，设备名称顺位相邻 (例如 `FIT101` 与 `FIT102` 编号差值为1)，视为在工艺管线中相邻。
5. **周期相似性规则**: 同一工艺段内，如果两个设备的时间序列周期性特征向量的**余弦相似度 (Cosine Similarity)** $\ge$ 设定的阈值 (`period_similarity_threshold`，默认 0.9)，则它们享有动态相似流态逻辑，建立连接。

*(所有边在生成的最后均会执行去重，保留绝对权重最大的边记录。也可通过 `--directed` 参数决定其为有向图或无向双向边)*

## 4. 时空图快照 (Temporal Snapshots)
在启用了 `--temporal_mode` 模式下，系统不仅构建一张全局静态图，而是通过给定的时间窗 (`temporal_window`) 和滑动步长 (`temporal_stride`) 切割时间序列，并在每一帧局部窗口重复以上特征抽取和相似性构边流程，形成一连串**含有时间维度的静态快照图组 (Temporal Heterogeneous Snapshots)**。

产出格式高度兼容 `PyG (PyTorch Geometric)` 的 `HeteroData` 结构格式（例如各个边关系被存储为单独的 numpy array 文件如 `edge_index__*.npy`, `edge_weight__*.npy`），可以直接应用于如 HTGNN (Heterogeneous GNN) 等高级时态-空间神经网络模型进行异常检测分析。
