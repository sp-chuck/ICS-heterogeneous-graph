# 第一版

## 1. 版本目标

本版完成了基于 SWAT 数据的三层异构图构建与时序训练入口打通。

核心目标：
- 以 AHGA 思路重构三层结构：CRP（虚构）- PLC（虚构）- 现场设备（list.txt）
- 同时构建三类边：hierarchy、physical、flow
- 支持时序快照导出（temporal snapshots）
- 提供 HTGNN 训练脚本，直接读取 snapshot 序列进行训练与验证

## 2. 已实现文件

- DataPreprocess_new.py
- train_htgnn.py
- temporal_snapshots/snapshots_metadata.json（时序快照汇总）

## 3. 数据与图构建实现（DataPreprocess_new.py）

### 3.1 输入

默认输入目录：/data/spz/data/swat_whole

使用文件：
- train.csv
- list.txt
- graph100.txt

### 3.2 三层节点结构

- 1 个 CRP 节点：CRP0（虚构）
- 6 个 PLC 节点：PLC1~PLC6（虚构）
- 现场设备节点：由 list.txt 对齐 train.csv 列后得到

PLC 分配规则：
- 按设备名中的数字首位映射：1xx->PLC1, 2xx->PLC2, ... , 6xx->PLC6

### 3.3 节点特征

设备节点特征（每设备 7 维）：
- mean, std, min, max, last, slope, wmean

其中 wmean 使用指数衰减加权（近期样本权重大）。
PLC/CRP 通过聚合下层节点特征得到。最终做全局 z-score 标准化。

### 3.4 边关系

- hierarchy：CRP<->PLC，PLC<->device
- physical：graph100 距离矩阵阈值建边（默认权重转换为 1 - d/threshold）
- flow：仅正常样本上计算 Pearson 相关，满足 |r|>=corr_threshold 建边

### 3.5 输出

静态图输出（out_dir）：
- node_mapping.csv
- hetero_node_features.csv
- edges_hierarchy.csv
- edges_physical.csv
- edges_flow.csv
- hetero_edges.csv
- x_crp.npy, x_plc.npy, x_device.npy
- edge_index__*.npy, edge_weight__*.npy
- hetero_meta.json
- preprocess_report.json, preprocess_report.md

时序图输出（temporal_mode）：
- temporal_snapshots/snapshot_00000/...（每个快照一套异构图文件）
- temporal_snapshots/snapshots_metadata.json
- temporal_snapshots/snapshots_metadata.csv

快照标签规则：
- 窗口内存在任意 attack=1 -> snapshot_label=1
- 否则 snapshot_label=0

## 4. 时序训练入口（train_htgnn.py）

### 4.1 模型实现

模型名称：HTGNN

当前实现结构：
- SnapshotEncoder：HeteroConv + SAGEConv（异构空间编码）
- TemporalEncoder：GRU（跨快照时间建模）
- Classifier：线性层输出二分类 logit

即：
- 每个快照先做异构图编码得到快照嵌入
- 再将快照序列输入 GRU
- 用序列末端状态预测当前序列标签

### 4.2 训练流程

- 从 temporal_snapshots 读取所有 snapshot
- 按 seq_len 组装连续序列样本
- 标签取序列最后一个快照的 snapshot_label
- 按时间顺序切分 train/val/test，避免时序泄漏
- 使用 BCEWithLogitsLoss 训练
- 输出 acc/precision/recall/f1/auc
- 早停并保存最佳模型

## 5. 当前运行状态（基于现有快照元数据）

从 temporal_snapshots/snapshots_metadata.json 可见：
- num_snapshots = 370
- window = 256
- stride = 128
- snapshot_labels: normal=370, attack=0

这意味着当前快照全为正常标签，训练阶段会出现类别单一问题（无法学习到攻击类别）。

---

## 6. 当前风险与限制

1. 类别不平衡/单一
- 当前时间窗口配置与数据切片下未覆盖攻击区间，导致 snapshot_label 全 0。

2. 依赖环境
- train_htgnn.py 依赖 torch_geometric。
- 若环境未安装 PyG，将无法导入 HeteroData/HeteroConv。

3. 训练有效性依赖快照标签
- 若 attack 标签始终缺失，AUC 等指标不具备解释意义。

---

## 7. 下一步建议（第二版候选）

1. 先修复时序样本覆盖
- 调整 temporal_window/temporal_stride 或切分范围，确保包含 attack 区间。

2. 增加标签分布保护
- 在预处理阶段增加检查：若 attack 快照数量为 0，直接报警并退出。

3. 增加训练稳健性
- 加入 class weight / focal loss。
- 增加按事件段评估指标（事件召回、告警延迟）。

4. 增加推理脚本
- 新增 infer_htgnn.py：加载 best model，对测试快照输出告警序列与事件级评估。

## 9. 第二版已实现改进（2026-04）

### 9.1 DataPreprocess_new 输出目录改进

- 新增参数：`--static_subdir`（默认 `static_graph`）
- 静态图输出统一写入：`out_dir/static_graph/`
- 时序输出保持不变：`out_dir/temporal_snapshots/`

静态输出（示例）：
- `static_graph/node_mapping.csv`
- `static_graph/hetero_node_features.csv`
- `static_graph/edges_hierarchy.csv`
- `static_graph/edges_physical.csv`
- `static_graph/edges_flow.csv`
- `static_graph/hetero_edges.csv`
- `static_graph/x_crp.npy, x_plc.npy, x_device.npy`
- `static_graph/edge_index__*.npy, edge_weight__*.npy`
- `static_graph/hetero_meta.json`
- `static_graph/preprocess_report.json, preprocess_report.md`

### 9.2 参照 DataPreprocess_Data 的关键预处理策略

在不替换第一版异构图时序主架构的前提下，新增旧版兼容模式：

- 新增参数：`--legacy_window_mode`
- 新增参数：`--legacy_device_indexes`
	- 默认值与旧版一致：`1,2,3,4,6,7,8,9,10,13,15,17,18,19,20,21,22,23,25,26,27,28,29,35,36,37,39,40,41,42,45,46,47,48,50`
- 新增参数：`--legacy_block_size`（默认 `100`）

legacy 模式下设备特征：
- 使用 `window_size=4300`、`balance_factor=4` 的指数加权
- 按 `legacy_block_size=100` 分段聚合
- 设备节点特征从 7 维切换为 43 维（与旧版聚合思想对齐）

快照标签规则仍保持：
- 窗口内任一 `attack=1`，则 `snapshot_label=1`

新增标签覆盖保护：
- 参数 `--fail_on_no_attack_snapshots`
- 若快照标签全为 0，可选择告警或直接中止

### 9.3 仅现场设备 ground truth（预处理 + 训练）

预处理端（每个快照）新增：
- `device_ground_truth.csv`
- `device_ground_truth.npy`

策略：
- 仅对 `device` 节点导出监督标签
- CRP/PLC 不导出监督标签

快照元数据新增字段：
- `num_device_nodes`
- `num_device_attack_labels`
- `device_attack_ratio`

快照总元数据 `snapshots_metadata.json` 新增：
- `device_ground_truth` 汇总统计

训练端（train_htgnn.py）新增：
- 参数 `--device_loss_alpha`（默认 `0.3`）
- 总损失：`L = L_snapshot + alpha * L_device`
	- `L_snapshot`：序列末端快照二分类损失
	- `L_device`：仅设备节点辅助损失

评估输出新增：
- `device_acc`
- `device_precision`
- `device_recall`
- `device_f1`
- `device_auc`
